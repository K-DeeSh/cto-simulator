# CTO Simulator: Ignore Problems

Браузерная мини-игра. Вы — CTO. На вас валятся проблемы.
Выживите до конца квартала.

## Быстрый старт

```bash
npm install
npm run dev
```

Открыть: http://localhost:5173

## Production-сборка

```bash
npm run build
# готовая сборка в папке dist/
npm run preview  # локальный превью prod-сборки
```

## Как играть

- 50 ходов = один квартал (40 на "Пятница вечер")
- На каждом ходу появляется карточка с проблемой
- Нажмите **Игнорировать** или **Делегировать**
- Следите за метриками компании
- Если любая из ключевых метрик упадёт до нуля — игра окончена
- Доживите до конца квартала — и узнайте свой CTO-архетип

## Метрики

| Метрика | Потеряли? |
|---|---|
| 🏢 Здоровье | Компания рухнула |
| 💪 Мораль | Вся команда уволилась |
| 💰 Бюджет | Деньги кончились |
| ⭐ Репутация | Публичный позор |
| 🗑️ Техдолг | Растёт при игнорировании |

## Архитектура

```
src/
├── types.ts              # все TypeScript-типы
├── data/
│   └── events.ts         # 45+ игровых событий
├── game/
│   ├── engine.ts         # чистая игровая логика (без React)
│   └── archetypes.ts     # расчёт итогового архетипа и счёта
├── components/
│   ├── StartScreen.tsx   # стартовый экран + рекорды
│   ├── GameScreen.tsx    # основной игровой экран
│   ├── EndScreen.tsx     # результат + архетип
│   ├── MetricsPanel.tsx  # панель метрик с анимацией
│   ├── ProblemCard.tsx   # карточка проблемы
│   ├── ActionButtons.tsx # кнопки Игнорировать / Делегировать
│   └── EventLog.tsx      # лента последних событий
├── App.tsx               # управление фазами игры + high scores
├── main.tsx
└── index.css             # всё стилевое, mobile-first
```

**Ключевые решения:**

- **Turn-based fast** — следующая карточка сразу после выбора, удобно на телефоне
- **Вероятностные исходы** — `riskChance` добавляет случайность, игра не детерминирована
- **Отложенные последствия** — `delayedEffect` в событии добавляет `PendingConsequence` в очередь; срабатывает через N ходов
- **Игровая логика отделена от UI** — `engine.ts` — чистые функции, без React
- **Никакого Redux** — `useState` в `App.tsx` достаточно для проекта такого размера
- **High scores** через `localStorage`, 5-10 записей
